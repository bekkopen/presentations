Kom i gang

I starten må du m.a. identifisere deg...

$ git config --global user.name "Mikael Vik"
$ git config --global user.email mikael.vik@gmail.com

Opprett lokalt repo og sjekk inn alt

$ git init
$ git add .
$ git commit -m "Første commit"

Fil-tilstander

Commit-flyt

  • Untracked: couldn't care less
  • Tracked
    • Uendra
    • Endra: må addes
    • Staga: WOHOOOO!!!

Bruk git status for å sjekke.. status

Alle filer som er staga blir med i neste commit

Så gjør vi så

Staging:

$ git add .          # add alle filer
$ git add css        # add alt i mappa css
$ git add css/*.css  # add alle css-filer i mappa css
$ git add -u .       # add endringer kun for tracka filer

Commit:

$ git commit -m  "Sjekker inn staga endringer"
$ git commit -am "Sjekker inn endringer i tracka filer"
$ git commit         # sjekk inn staga filer, editor for message
$ git commit --amend # endre på siste commit

git log

Hovedverktøyet for å se på commit-historikken. Ekstremt kraftig. 

$ git log
$ git log -1
$ git log -10
$ git log --oneline

$ git log --graph --pretty=format:'%h -%C(red)%d%Creset %s <%an>' -3
# lag et alias for finere log!!

Hvordan lagrer Git the shit?

Git thinks of its data more like a set of snapshots of a mini filesystem. Every time you commit, or save the state of your project in Git, it basically takes a picture of what all your files look like at that moment and stores a reference to that snapshot.

En git-commit:

  • Er et snapshot av filsystemet ditt
  • Ikke kun endringer fra forrige commit
  • Vet tilstanden til _alle_ filene
    • Lagrer kopi av alle endra filer
    • Linker til forrige versjon for uendra filer
  • Peker til forrige commit
  • ... og har chekcsum på ALT.

Finn et bilde

git branch

Lettvekts. Kun en peker til en commit-sha.

$ git branch greier          # opprett ny branch 'greier'
$ git checkout greier        # sjekk ut branchen

$ git checkout -b nye-greier # opprett og sjekk ut ny branch 'nye-greier'

$ git checkout master        # bytting av branch
$ git checkout nye-greier

$ find .git/refs/heads -type f
$ cat .git/HEAD

HEAD: peker alltid til siste commit i branchen du står i

git merge -ff

Fast forward

Flytter pekeren

Dette er den enkleste forma for merging


$ git co -b iss53
$ git commit -am "Issue-53 fiksa"
$ git co master
$ git merge iss53 # flytt master-pekeren fram

git merge --no-ff

No fast forward

Lager en merge commit

Viser eksplisitt at commits har oppstått i en anna branch 


$ git co -b iss53
$ # flere commits
$ git co master
$ git pull --ff-only
$ git merge iss53 # oppretter en merge-commit

Måten Git lagrer commits på er _nettopp_ det som gjør merging så kraftig...

Remotes

Når du skal samarbeide om kode må du legge til minst én remote

$ git remote add origin git@github.com:mikaelvik/nyoppretta-repo.git
$ git push origin master

$ git push
$ git pull # helst med flagget --ff-only

# git pull er faktisk to operasjoner:
$ git fetch
$ git merge origin/<branch-navn>

Eller lær frå dei beste...

$ git clone git@github.com:bekkopen/presentations.git

Tacklebox

Alias

Livsnødvendig med haugevis av alias for å komme gjennom dagen, i live.
Mikael Vik
$ git config --global alias.co checkout
$ git config --global alias.br branch
$ git config --global alias.ci commit
$ git config --global alias.st status

$ git config --global alias.softundo=reset --soft HEAD~1

$ git config --global alias.lg=log --graph --pretty=format:'%Cred%h%Creset -%C(yellow)%d%Creset %s %Cgreen(%cr) %C(blue)<%an>%Creset' --abbrev-commit --date=relative
